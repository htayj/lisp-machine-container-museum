# Curated MIT CADR runtime screenshots

This directory contains twenty-two reviewed runtime captures selected from thirteen
provenance-bearing System 303 sessions: twenty-one exact full framebuffers and one
documented crop that retains a complete application window. They are runtime observations, not
manual illustrations and not claims about every CADR, application, or debugger release.

## Rights status

These twenty-two PNGs are included for the museum's criticism, comment, scholarship, and
research under the fair-use limitation in
[17 U.S.C. § 107](https://www.copyright.gov/title17/92chap1.html#107). This is an
asset- and use-specific conclusion, not a statement that every screenshot of every
computer system is automatically free to publish.

The repository-wide selection criteria and reassessment process are documented in
the [screenshot publication rights review](../../screenshot-publication-rights-review.md).

The Copyright Office explains that copyrightable screen displays generated by a
program may be covered with the program in
[Circular 61](https://www.copyright.gov/circs/circ61.pdf). A screenshot therefore is
not automatically outside copyright merely because a researcher captured it. The
publication basis here is the limited documentary use assessed below.

The present use is narrowly documentary: each image appears beside analysis of the
exact application or recovery state it proves; the set contains twenty-two low-resolution still
frames from a much larger interactive system; the images cannot substitute for the
software, load band, source, fonts, or manuals; and no market for licensing these
System 303 observations was identified. The Ninth Circuit held the use of a handful of actual
software screenshots fair in
[*Sony Computer Entertainment America v. Bleem*, 214 F.3d 1022](https://www.copyright.gov/fair-use/summaries/sonycomputer-bleem-9thcir2000.pdf),
and the Second Circuit treated images used as historical artifacts in explanatory
context as fair in
[*Bill Graham Archives v. Dorling Kindersley*, 448 F.3d 605](https://www.copyright.gov/fair-use/summaries/billgraham-dorling-2dcir2006.pdf).
Fair use remains a case-specific U.S. doctrine.

The displayed window, ZWEI, Zmacs, mode, redisplay, and who-line source files identify
Massachusetts Institute of Technology copyrights. The Help capture includes a single
functional dispatcher screen generated by
[`zwei/doc.lisp`](https://tumbleweed.nu/r/sys/file?ci=4df393c68d7f083ce42d5c377039d26043cc18a9&name=zwei%2Fdoc.lisp&ln=1-132).
The restrictive historical LMI notice elsewhere in the LM-3 tree belongs to the
separate
[`tape` subsystem](https://tumbleweed.nu/r/sys/file?ci=4df393c68d7f083ce42d5c377039d26043cc18a9&name=tape%2Fcopyright.text&ln=1-12),
which is not displayed in these images. Its presence is a reason not to infer a
repository-wide license, but it does not make these Zmacs pixels into Tape-system
content. The earlier public System 46 release has a
[separate BSD-style license](https://github.com/mietek/mit-cadr-system-software/blob/8e978d7d1704096a63edd4386a3b8326a2e584af/src/LICENSE);
its pinned
[release README](https://github.com/mietek/mit-cadr-system-software/blob/8e978d7d1704096a63edd4386a3b8326a2e584af/src/README)
says MIT licensed that software and credits Tom Knight's work with MIT's patent and
technology-licensing office to secure release permission. That provenance is specific
to the released System 46 tapes; this catalog does not extend their license to the
maintained System 303 tree.
The raster lettering visible in the frames does not distribute a font program; U.S.
Copyright Office regulations also exclude
[“typeface as typeface”](https://www.copyright.gov/title37/202/37cfr202-1.html)
from copyright registration. Neither point substitutes for the fair-use analysis of
the screen as a whole.

The PNGs are excluded from any present or future project-wide content or software
