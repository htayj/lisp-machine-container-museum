# ZWEI and Zmacs on the MIT CADR and LM-3

ZWEI is the reusable editing system; Zmacs is its general file, buffer, and Lisp
development editor. The distinction is visible in the source: the standard ZWEI
command table, editing primitives, redisplay, modes, and editor streams stand
below a Zmacs command-table overlay and Zmacs-specific buffer, file, source, and
window machinery.

This page covers architecture, features, modes, and observed behavior. The
[binding companion](zwei-zmacs-keybindings.md) gives release-bounded direct-key
inventories and links the complete public System 46 named-command output.

## Two public evidence sets

### MIT CADR System 46 snapshot

The museum pins `mietek/mit-cadr-system-software` at revision
[`8e978d7`](https://github.com/mietek/mit-cadr-system-software/tree/8e978d7d1704096a63edd4386a3b8326a2e584af).
Its `src/nzwei/` directory contains ZWEI structures, redisplay, commands, modes,
mouse support, Dired/Mail support, generated QFASL files, self-documentation, and
an exact generated command listing. It also contains Zmacs structures and tags
which name an original `NZWEI; ZMACS` file, but that Zmacs implementation file
is absent from the public snapshot.

This is an important preservation boundary. The System 46 standard ZWEI map and
built help output are directly inspectable. The complete Zmacs overlay cannot be
reconstructed from the public source tree alone, so this museum does not silently
substitute a later source file and label it System 46.

The system declaration names 31 active modules, of which 25 source files are
present. Those 25 total 635,071 bytes with aggregate manifest SHA-256
`fb2f381f464aed8f05d2c4be8c9113c2b3e63a0ad9fe66fa9088a3e95a4791a1`.
A non-evaluating audit found 345 literal concrete `DEFCOM` names and 17 mode
entry commands generated by 11 major- and 6 minor-mode declarations: 362
source-defined command symbols in the preserved subset. Its 450 expanded
literal `SET-COMTAB` events are assignments, not 450 distinct live keys.

The tracked [System 46 generated command listing](../assets/mit-cadr-online-help/standalone/_comnd.1)
records 348 named ZWEI/Zmacs commands and the direct bindings exposed when that
artifact was generated. `emacs.comdif` and `nzwei.comdif` in the same directory
record contemporary compatibility and change notes. The
[on-line help recovery article](online-help-and-documentation-recovery.md)
explains how those artifacts and source declarations were recovered.

## How ZWEI represents editing

Text is an interval made of line objects. Buffer pointers, or BPs, identify a
line and position and can move as text changes. Windows display intervals and
retain their own point, mark, and display state. This separation permits two
windows to show the same buffer at different positions while sharing changes.

The command loop dynamically binds the current editor, window, interval, point,
mark, mode state, command tables, histories, syntax tables, and streams. A
command returns a redisplay degree, allowing a character insertion, a line
change, and a full window reformat to request different amounts of work.

The System 303 system declaration shows the scale of the subsystem. Separate
modules implement:

- structures, macros, screen integration, redisplay, and text primitives;
- movement, insertion/deletion, indentation, Lisp parsing, and search;
- kill, minibuffer, and undo histories;
- fonts, keyboard macros, modes, mouse commands, and self-documentation;
- general commands, files, Dired and directory comparison, spelling, patching,
  possibilities/warnings, and editor streams;
- Zmacs section, buffer, file, and top-level behavior.

## Command tables are composed, not copied

A ZWEI command table stores direct characters, mouse gestures, a named-command
alist, an optional parent table, and an optional Control-key indirection. A leaf
can also be an alias, prefix table, or menu command. Lookup follows those links
until it finds a command or an undefined slot.

System 303 builds Zmacs by composition:

1. `STANDARD-COMTAB` defines general editing;
2. `STANDARD-CONTROL-X-COMTAB` supplies the standard prefix;
3. `ZMACS-COMTAB` overrides editor-specific keys and inherits the standard
   table;
4. `ZMACS-CONTROL-X-COMTAB` overrides files, buffers, and windows and inherits
   the standard prefix;
5. major and minor modes can put local command tables in front of those layers.

This explains a historical bug discussion preserved in `bugs.old`: mode-aware
Zmacs should consult the mode table, the Zmacs table, and the ZWEI table. It also
explains why counting only literal forms in `comtab.lisp` is not a count of the
effective runtime map.

## Feature inventory

### Editing and navigation

- insertion, overwrite, quoting, transposition, deletion, kill/yank history,
  registers, and keyboard macros;
- motion and selection by character, line, page, screen, word, sentence,
  paragraph, s-expression, list, definition, and saved point;
- matching and manipulation of list delimiters, parenthesis-balance checks,
  indentation, code formatting, and Lisp-aware case conversion;
- active regions with underline or reverse-video feedback;
- undo and redo in the System 303 source, including quick key bindings;
- mouse marking, region movement, kill/yank, and indentation operations.

### Search, transformation, and generated result buffers

- forward and reverse incremental search, Lisp-aware match search, literal and
  query replace, multi-string replacement, and tag-table-wide operations;
- occurrence lists, matching-line filters, sorting, reversing, and keyboard-
  macro-driven transforms;
- “possibility” buffers used to visit a sequence of search results, warnings,
  callers, or other source locations;
- source comparison and merging.

### Buffers, files, windows, and directories

- named buffer history, create/select/list/edit/kill/revert/rename operations,
  read-only state, and modified-buffer review;
- file find/visit/read/write/save/append/prepend/copy/rename/delete/undelete,
  file properties, directories, and background file finding;
- one, two, and several-window layouts, another window, window growth, two
  views of one region, and buffers chosen for each pane;
- Dired and directory-difference editors with their own major modes and local
  commands.

### Lisp and system development

- evaluate or compile regions, definitions, buffers, changed sections, and
  files; compile-and-load and FASL update;
- edit definitions, callers, object users, methods, combined methods, and flavor
  relationships;
- argument lists, documentation, variable description, macro expansion,
  disassembly, trace/untrace, and package/readtable/base attributes;
- tag tables over files, buffers, packages, or systems;
- compiler-warning navigation and patch creation, resumption, completion, and
  release operations;
- a ZTOP mode in which a ZWEI buffer and editor stream feed a Lisp top-level,
  preserving editable input history and package state.

### Text production and appearance

- Text, Bolio, and TeX editing; paragraph filling; comment filling; column and
  prefix control; spelling correction; and word abbreviations;
- font selection and character/word/region font changes;
- text-justifier font and underline control;
- printing buffers, regions, files, and all buffers.

### Help and customization

The Help dispatcher can give a basic introduction, describe a key or named
command, search command names, show where a command is bound, describe
variables, repeat a help query, and show recent input. `Meta-X` normally offers
the active named-command alists; `Control-Meta-X` can request any loaded ZWEI
command. Users can install functions, keyboard macros, and mouse macros on keys,
and can edit the implementation of the command installed on a key.

Self-documentation walks the command-table structure rather than consulting a
fixed wall chart. The tracked System 46 `_comnd.1` file is therefore evidence of
a particular built environment, while the source tables establish how the list
was produced.

### Editor streams, ZDT, and ZTOP

System 303 also contains two interactive Lisp embodiments of ZWEI. They share
the editing machinery but are not alternate names for Zmacs:

- `ZDT` initializes the generic editor-stream command table, creates a single
  `EDITOR-TOP-LEVEL` window if necessary, and selects or deselects it. The
  source also registers that editor top level on the `Top-L` system key with
  the label `LISP(Edit)`. ZDT has no independent fixed keymap; it uses the
  eight-cell editor-stream overlay listed in the
  [binding reference](zwei-zmacs-keybindings.md#editor-stream-and-keyboard-macro-mover-tables).
- ZTOP is a Zmacs-buffer Lisp top level. Its major mode installs pre- and
  post-command hooks, editable input history, an eight-cell local table, and a
  mode-line state that distinguishes reading, editing, and waiting for explicit
  activation.

The generic editor-stream command hook normally submits buffered input
automatically when point remains at the end after eligible input. If point has
moved away, Require Activation mode can defer submission until `End` or
`C-Return`. This state-dependent behavior is visible in `stream.lisp` but would
be lost in a flat command-name or key list.

## Modes in the pinned trees

Modes are executable configurations: they set syntax, indentation, variables,
and local command tables. They are not only labels on the mode line.

| Evidence set | Major modes in the inspected source | Minor modes in the inspected source |
| --- | --- | --- |
| System 46 source-present subset | Dired, Mail, Lisp, MIDAS, Text, Bolio, Fundamental, PL1, Electric PL1, TECO, Macsyma | Atom Word, Emacs, Auto Fill, Overwrite, Word Abbrev, Electric Shift Lock |
| System 303 active subsystem | Dired, Edit Buffers, directory differences, Lisp, Scheme, T, MIDAS, Text, Bolio, Fundamental, PL1, Electric PL1, C, TECO, Macsyma, TeX, Possibilities, Warnings, ZTOP | Mail, Return Indents, Atom Word, Emacs, Any Bracket, Auto Fill, Overwrite, Word Abbrev, Electric Shift Lock, Electric Font Lock, Require Activation Mode 1, Require Activation Mode |

The table deliberately includes application modes defined outside the general
modes file. Dired, directory differences, Edit Buffers, Possibilities, Warnings,
Mail, and ZTOP embody task-specific editor state rather than merely selecting a
file language.

## What the manuals and generated help do not prove

- A command name in generated help proves that the command was registered in
  that built environment; it does not prove that every dependency was usable.
- A literal key form proves an initial table entry. Parent tables, modes,
  aliases, patches, and user installation determine the effective binding.
- The System 46 tags prove that a Zmacs source file existed when the tag table
  was made. They do not recover the absent file's contents.
- Six System 46 binding targets are absent from the source-present command set.
  Two are explicitly unconverted TECO operations; three are named in tags from
  missing modules; the Dired complement-no-delete target remains an open
  question. A configured cell is not proof that its target loaded successfully.
- In System 303, `*ZMACS-BUFFER-NAME-READING-COMTAB*` is declared but never
  initialized or referenced by the exact active corpus. It is evidence of stale
  or unfinished state, not an active table merely because its variable exists.
- System 303's extended-pattern finite-state search explicitly signals an error
  when asked to run backward. Ordinary reverse string search is a separate path;
  the limitation is specifically complex backward extended search.
- Dired's short `E` and `V` labels conceal type-sensitive behavior. On a
  directory entry, Edit enters another Dired view and View opens a directory
  viewer rather than treating the entry as a text file.
- A fresh Mail composition selects Text mode and enables the Mail minor mode,
  but the retain-existing-contents path skips that reset. A retained Mail buffer
  can therefore preserve prior editor state.
- Edit Buffers documents `U` as canceling all requested operations, but the
  inspected implementation writes column zero three times rather than clearing
  the three operation columns. Static analysis therefore suggests that some
  save, revert, or print marks can survive. This is a probable implementation
  bug, pending runtime verification.
- ZWEI's reusable editor streams, Dired, ZTOP, and Mail modes show multiple
  editor embodiments. They are not evidence for four independent editor
  lineages.
