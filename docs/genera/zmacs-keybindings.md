# Genera 8.5 Zmacs keybindings

This reference describes every configured direct entry in the inspected
standard Zwei, Zmacs, and `Control-X` source tables; every initialized base
minibuffer, pathname, FQUERY, extended-search, and macro-mover context; and the
complete local deltas for the inspected general and task modes. It contains
factual binding names and original summaries, not proprietary source or copied
manual text. All machine-derived Genera output remains ignored.

See [Zmacs in Symbolics Genera](zmacs.md) for architecture, feature coverage,
runtime observations, artifact identities, and rights boundaries. See the
[named-command audit](zmacs-named-commands.md) for the counts, functional
categories, and installation semantics of the 277 standard, Zmacs, and explicit
CP source candidates; their exact ordered arrays remain ignored local evidence.

## Scope and notation

- `C-`, `M-`, `S-`, and `H-` mean Control, Meta, Super, and Hyper; `Sh-` means
  Shift where case is significant.
- Named keys such as `Cut`, `Copy`, `Paste`, `Undo`, `Redo`, `Scroll`, `Find`,
  `Complete`, `Help`, and `Abort` are Genera characters, not guessed PC keys.
- A range is stated once and covers every cell generated by the source form.
  In `SET-COMTAB`, the second number is a count, not an endpoint; these bare
  source numerals are read in Base 8. The three standard ranges `(0 200)`,
  `(301 310)`, and `(312 317)` therefore
  expand to octal `000` through `177`, `301` through `610`, and `312` through
  `630`. Later specifications win where ranges or individual cells overlap.
- Commented-out entries are not bindings. Explicitly undefined local entries
  are listed because they shadow inheritance.
- The source configures 212 literal standard entries plus `C-X`, 40 standard
  prefix entries, 13 Zmacs direct entries plus `C-X` and a mouse menu, and 22
  Zmacs-prefix entries. Counts are constructor entries, not runtime-active keys:
  ranges, inheritance, aliases, modes, patches, optional systems, and user files
  alter the effective environment.
- The subordinate contexts add 50 literal configured cells: 21 minibuffer,
  pathname, and FQUERY cells; 28 extended-search cells; and one synthetic
  keyboard-macro-mover event. Five are explicit undefined shadows.

## Keyboard-macro mover

`comg.lisp.~96~` creates a transient sparse table with one synthetic entry: the
internal `BLIPS:a/` sentinel exits Keyboard Macro Mover. It is not a physical
key. Before use, the sparse table inherits the current active comtab, so a
recorded mover retains the editing context in which its motion commands were
entered. The source file is 17,848 bytes with SHA-256
`66007d528c90407fb162df152e6dfc383282981c0162abd67ebd3c6702784043`.

## Source and runtime status

The binding facts come from the licensed source revisions and hashes recorded in
[the Zmacs article](zmacs.md#evidence-and-rights-boundary). The fresh 8.5 Xvfb
session confirmed `Select E`, `Help`, `Control-X 3`, `Control-X Control-B`, and
the presentation/mouse behavior described there. It did not execute every leaf,
